# Анимация для ЛР1. Самопроверка контрольной суммы сегмента `.text`

## Описание

В проекте реализована демонстрация механизма защиты приложения от модификации (патча) с использованием самопроверки контрольной суммы.

Алгоритм основан на вычислении контрольной суммы исполняемого кода программы (сегмент `.text`) и сравнении её с эталонным значением при каждом запуске.

## Алгоритм работы

Алгоритм работает следующим образом:

1. Определяется виртуальный адрес начала сегмента `.text`.
2. Определяется размер сегмента `.text`.
3. Из памяти процесса считывается блок данных, соответствующий сегменту `.text`.
4. Вычисляется контрольная сумма данного блока.
5. Полученная сумма сравнивается с заранее сохранённой эталонной контрольной суммой.
6. Если значения совпадают — программа считается неизменённой.
7. Если значения не совпадают — фиксируется факт модификации (патча) и выводится предупреждение.

## Что именно проверяется

Проверяется сегмент `.text`, так как он содержит машинный код программы. Любое изменение бинарника (патч, вставка инструкций, удаление кода) приводит к изменению содержимого этого сегмента и, соответственно, к изменению контрольной суммы.

## Анимация

В анимации показано:

- как определяется начало сегмента `.text`;
- как определяется его размер;
- как происходит вычисление контрольной суммы;
- как выполняется сравнение с эталонным значением;
- два сценария:
  - контрольные суммы совпадают (программа не изменена);
  - контрольные суммы не совпадают (обнаружена модификация).

## Вывод

Самопроверка контрольной суммы сегмента `.text` позволяет обнаруживать изменение исполняемого кода приложения. Метод прост в реализации и эффективен для выявления патчинга, однако может быть обойдён при достаточном уровне подготовки нарушителя.
